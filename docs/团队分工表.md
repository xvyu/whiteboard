## 二、详细分工说明

### 1. 陈超滨（34%）— 第三阶段：协作增强与存储

| 编号 | 任务内容 | 涉及技术/模块 |
|:---:|:---|:---|
| 1 | 数据库集成：PostgreSQL + Redis 5.x 双存储架构设计与实现 | SQLAlchemy、psycopg2、redis-py |
| 2 | 实时状态双写：每次绘图操作同时写入 Redis 和 PostgreSQL | 双写策略、事务一致性 |
| 3 | 定时快照存储：每 5 分钟自动保存画板快照到数据库 | APScheduler、asyncio |
| 4 | 服务重启状态恢复：启动时从最新快照加载房间状态 | 反序列化、全量状态推送 |
| 5 | 聊天系统：实时聊天功能，消息持久化到数据库，支持删除 | WebSocket、XSS过滤 |
| 6 | 房间自动清理：定时扫描空房间，释放内存和数据库资源 | asyncio 定时器 |
| 7 | 房间密码与房主权限：房主设置密码、踢人功能实现 | bcrypt 密码加密 |
| 8 | 用户认证系统：JWT 登录/注册、游客模式、Token 鉴权 | PyJWT、Werkzeug |
| 9 | 存档系统：保存/加载画板存档，列表显示与悬停预览 | 缩略图生成 |
| 10 | 页面刷新自动重连与退出房间功能 | WebSocket 断线重连 |

---

### 2. 张兆嘉（33%）— 第二阶段：体验完善

| 编号 | 任务内容 | 涉及技术/模块 |
|:---:|:---|:---|
| 1 | 图形对象选中与编辑：点击选中、移动、缩放、删除、改色 | Canvas 2D 事件处理 |
| 2 | 颜色/粗细面板优化：取色器、预设色板、粗细滑动条 | 前端 UI 交互 |
| 3 | 断线重连与状态恢复：自动重连后同步全量状态 | WebSocket reconnection |
| 4 | 性能优化：画笔点阵实时渲染、增量更新、合并保护 | RAF 帧率控制 |
| 5 | 移动端触摸优化：触摸绘制、侧边栏收起、工具栏横向滚动 | Touch Events |
| 6 | 自定义昵称：用户可修改显示昵称并广播 | WebSocket 消息 |
| 7 | 导出为图片：画板内容保存为 PNG 截图 | Canvas toDataURL |
| 8 | 文字工具：点击添加文本、编辑内容、拖动位置 | Canvas text rendering |
| 9 | 箭头/直线工具完善：拖拽生成直线与箭头模式 | 几何计算 |
| 10 | 网格显示：切换网格背景辅助绘图 | Canvas 网格绘制 |

### 3. 熊乙鸣（33%）— 第一阶段：MVP 核心功能

| 编号 | 任务内容 | 涉及技术/模块 |
|:---:|:---|:---|
| 1 | WebSocket 基础通信：搭建 FastAPI 原生 WebSocket 服务，连接/断开/心跳/房间管理 | FastAPI WebSocket |
| 2 | 画笔工具：自由绘制、颜色/粗细选择、三种笔刷模式（画笔/标记笔/荧光笔） | Canvas 绘图 API |
| 3 | 操作序列化与广播：定义绘图操作 JSON 格式，实现发送与接收广播 | JSON 序列化 |
| 4 | 房间管理：创建/加入房间（URL 路由），内存存储画板状态 | 内存数据结构 |
| 5 | 新用户状态同步：加入时获取全量图形对象并完整渲染 | WebSocket init 消息 |
| 6 | 基本图形工具：矩形、圆形、三角形、直线、箭头（拖拽生成） | Canvas 几何绘制 |
| 7 | 橡皮擦：圆形区域擦除，支持调整半径 | Canvas 合成模式 |
| 8 | 撤销/重做功能：全局撤销+重做+箭头光标删除 | 操作历史栈 |
| 9 | 用户列表与在线人数：显示昵称、在线数，支持修改昵称 | 房间状态管理 |
| 10 | 实时光标显示：其他用户光标位置及颜色区分 | WebSocket cursor_move |
